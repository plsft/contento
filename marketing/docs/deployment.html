Docs Cloud Hosting

Cloud-Hosted. No Deployment Required.

Contento manages all infrastructure. You focus on content.

What Contento Manages

Contento is a fully managed cloud platform hosted at contentocms.com. There are no servers to provision, no containers to deploy, and no databases to manage.

InfrastructureDetails
Application serversAuto-scaled, zero-downtime deployments
DatabaseManaged PostgreSQL with automated backups
CacheManaged Redis for job queues and real-time streaming
CDNCloudflare edge caching for generated pages
SSLAutomatic SSL certificates for all subdomains via Cloudflare
AI inferenceManaged AI provider connections with rate limiting
BackupsDaily automated backups with 30-day retention
Monitoring24/7 uptime monitoring and alerting

Getting Started

The only thing you need to install is the CLI:

$ npm install -g contento
$ contento login

That is it. No Docker, no docker-compose, no Nginx, no database setup. See the Install CLI guide for full details.

Your Only Responsibility: DNS

The one thing you configure outside of Contento is a DNS CNAME record pointing your subdomain to pseo.contentocms.com. Contento handles everything else, including SSL certificate provisioning and CDN configuration.

See DNS & Domains for the setup guide.

# The only external configuration you need
Type:    CNAME
Name:    articles
Target:  pseo.contentocms.com
Proxy:   Proxied (orange cloud)

Uptime & SLA

  • 99.9% uptime SLA on Growth and Scale plans
  • Global edge network via Cloudflare for fast page delivery
  • Status page at status.contentocms.com
Coming from self-hosted? If you previously deployed Contento with Docker, you can migrate your data to the cloud platform. Contact support for a migration guide.
