Docs CLI Reference

CLI Reference

Every command, flag, and configuration option for the Contento CLI.

Installation

$ npm install -g contento
$ contento --version
  contento/1.0.0

Requires Node.js 18+ and npm 9+. See Install CLI for detailed setup.

Global Flags

FlagDescription
--jsonOutput as JSON (machine-readable). Required for agent integration.
--api-url <url>Override the API base URL. Default: https://app.contentocms.com
--no-colorDisable colored output
-v, --verboseShow detailed request/response logging
-V, --versionPrint version number
-h, --helpShow help for any command

Authentication

contento login

Authenticate interactively. Opens a browser for OAuth or prompts for email/password.

$ contento login

  ? Email: admin@example.com
  ? Password: ********
  ✓ Logged in. Credentials saved to ~/.contento/config.json

contento logout

Remove stored credentials.

$ contento logout
  ✓ Logged out. Credentials removed.

contento whoami

Show the currently authenticated user.

$ contento whoami
  admin@example.com (DevTools Resources)

Non-Interactive Authentication

For CI/CD and AI agents, set the CONTENTO_API_KEY environment variable:

$ export CONTENTO_API_KEY=ck_live_abc123...
$ contento whoami
  admin@example.com (API key auth)

Projects

contento projects list

List all projects in your account.

$ contento projects list

  ID            Name                 Subdomain                  Status   Pages
  proj_8f3a...  DevTools Resources   resources.example.com      active   248
  proj_2b1c...  Marketing Blog       blog.example.com           active   120

contento projects create

Create a new pSEO project.

FlagDescription
--name <name>Project name (required)
--domain <domain>Root domain (e.g., example.com)
--subdomain <sub>Full subdomain (e.g., resources.example.com)
--backlink-url <url>Backlink destination URL
--backlink-text <text>Backlink anchor text
 
$ contento projects create \
    --name "DevTools Resources" \
    --domain example.com \
    --subdomain resources.example.com \
    --backlink-url "https://example.com" \
    --backlink-text "Example Inc"

  ✓ Project created: proj_8f3a...
  Status: pending_dns
  Add a CNAME record: resources -> pseo.contentocms.com

contento projects status

Show detailed project status including DNS, SSL, and page counts.

$ contento projects status --project proj_8f3a...

contento projects verify

Trigger immediate DNS verification for a project.

$ contento projects verify --project proj_8f3a...

contento projects chrome

Upload custom chrome (header, footer, CSS) from files or inline.

FlagDescription
--project <id>Project ID (required)
--header <file>Path to header HTML file
--footer <file>Path to footer HTML file
--css <file>Path to custom CSS file
--header-inline <html>Inline header HTML string
--footer-inline <html>Inline footer HTML string

Niches

contento niches list

List available niches. Filter by category.

$ contento niches list --category "Technology"

  ID            Name               Category     Subtopics  Type
  niche_01...   Developer Tools    Technology   20         system
  niche_02...   DevOps             Technology   18         system
  niche_03...   Cloud Computing    Technology   22         system

contento niches show

Show full niche context (audience, pain points, subtopics).

$ contento niches show niche_01... --json

contento niches fork

Create a custom copy of a system niche for editing.

$ contento niches fork niche_01... --name "Custom DevTools"

  ✓ Forked "Developer Tools" as "Custom DevTools" (niche_4f2a...)
  Edit subtopics and context at: contento niches edit niche_4f2a...

Schemas

contento schemas list

List all available content schemas.

$ contento schemas list

  Slug             Name              Title Pattern
  idea-list        Idea List         {count} {subtopic} Ideas for {year}
  comparison       Comparison        {subtopic} vs Alternatives: Comparison
  how-to           How-To Guide      How to {subtopic}: Complete Guide
  best-of          Best-Of List      Best {subtopic} Tools & Resources
  faq              FAQ               {subtopic} FAQ: Common Questions

contento schemas show

Show full schema definition including SchemaJson and PromptTemplate.

$ contento schemas show idea-list --json

Collections

contento collections list

List collections for a project.

$ contento collections list --project proj_8f3a...

contento collections generate

Create a collection and start AI generation. This is the core command.

FlagDescription
--project <id>Project ID (required)
--schema <slug>Schema slug (e.g., idea-list) (required)
--niches <ids>Comma-separated niche IDs (required)
--workers <n>Concurrent AI workers. Default: 5
--jsonOutput NDJSON progress stream
 
$ contento collections generate \
    --project proj_8f3a... \
    --schema "idea-list" \
    --niches "niche_01...,niche_02..." \
    --workers 10

  Generating 40 pages across 2 niches...
  [00:26] ████████████████████ 40/40 (100%)
  Validation: 38 passed, 2 retried, 0 failed

With --json, outputs NDJSON for agent consumption:

$ contento collections generate --project proj_8f3a... --schema idea-list --niches niche_01... --json

{"type":"progress","current":0,"total":40,"done":false}
{"type":"page","id":"page_01...","status":"passed","title":"100 CI/CD Ideas"}
{"type":"progress","current":20,"total":40,"done":false}
{"type":"progress","current":40,"total":40,"done":true}
{"type":"done","passed":38,"retried":2,"failed":0}

Publishing

contento publish

Publish generated pages with one of four modes.

FlagDescription
--collection <id>Collection ID (required)
--mode <mode>immediate, batched, scheduled, or manual
--batch-size <n>Pages per batch (for batched/scheduled modes)
--cron <expr>Cron expression (for scheduled mode)
 
$ contento publish --collection col_9d2f... --mode immediate

contento publish history

View publish batch history for a project.

$ contento publish history --project proj_8f3a...

contento pages approve

Approve pages for manual publishing mode.

$ contento pages approve page_01... page_02... page_03...

contento pages unpublish

Remove a published page from the live site.

$ contento pages unpublish page_01...

Analytics

contento analytics summary

Project-level analytics summary (clicks, impressions, CTR, position).

$ contento analytics summary --project proj_8f3a...

contento analytics top-pages

Top pages ranked by performance.

FlagDescription
--project <id>Project ID (required)
--limit <n>Number of pages to show. Default: 20
--sort <field>Sort by: clicks, impressions, ctr, position
 
$ contento analytics top-pages --project proj_8f3a... --limit 10 --sort clicks

contento analytics zero-traffic

Pages with zero clicks for 60+ consecutive days.

$ contento analytics zero-traffic --project proj_8f3a...

contento analytics export

Export analytics data as CSV.

FlagDescription
--project <id>Project ID (required)
--format <fmt>csv or json. Default: csv
--output <file>Output file path. Default: stdout
 
$ contento analytics export --project proj_8f3a... --output analytics.csv

contento analytics connect-gsc

Connect Google Search Console via OAuth.

$ contento analytics connect-gsc --project proj_8f3a...

contento analytics sync

Trigger a manual GSC data sync (normally runs daily).

$ contento analytics sync --project proj_8f3a...

Configuration

The CLI stores configuration in ~/.contento/config.json:

{
  "apiUrl": "https://app.contentocms.com",
  "token": "ck_live_abc123..."
}

Environment Variables

VariableDescription
CONTENTO_API_KEYAPI key for non-interactive auth (overrides stored token)
CONTENTO_API_URLOverride API base URL (overrides config file)
NO_COLORDisable colored output (standard convention)

Precedence: environment variables > config file > defaults.
