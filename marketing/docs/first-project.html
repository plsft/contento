First Project

Create a project, select niches, generate pages, and publish — end to end.

Prerequisite: Make sure the CLI is installed. See Install CLI if you have not set it up yet.

Step 1: Authenticate

contento login

Opening browser for authentication...
Authenticated as george@example.com

# Verify you're logged in
contento whoami
george@example.com (Growth plan)

Step 2: Create a Project

A project defines your subdomain, branding, and back-link configuration:

contento projects create \
    --name "My First Project" \
    --domain "example.com" \
    --subdomain "resources.example.com"

  Project created: proj_8f3a...
  Status: pending_dns
  Subdomain: resources.example.com

  Add this DNS record:
    Type:   CNAME
    Name:   resources
    Target: pseo.contentocms.com

The project starts in pending_dns status. You need to add a CNAME record before pages can be served.

Step 3: Configure DNS

Add a CNAME record in your DNS provider:

TypeNameTargetProxy
CNAMEresourcespseo.contentocms.comProxied (Cloudflare)

Contento polls DNS every 5 minutes. Once verified, the project transitions to active. Check status:

contento domains verify --project proj_8f3a...

  resources.example.com
    DNS:  ✓ verified
    SSL:  ✓ active (Cloudflare)
    Status: active
Tip: You can proceed with generation while DNS is propagating. Pages will be stored and ready to serve once DNS is verified.

Step 4: Browse Niches

Contento ships with 150+ niches. Browse them to find your target audience:

contento niches list --category "Technology"

  Name                 Category      Subtopics
  ──────────────────────────────────────────────
  Developer Tools      Technology    20
  DevOps              Technology    22
  Cloud Computing      Technology    16
  ... 22 more

Note the niche IDs — you will use them when generating. Use --json to get IDs in machine-readable format.

Step 5: Choose a Schema

Pick a content type for your first batch. The idea-list schema is a good starting point:

contento schemas list

  Slug             Title Pattern
  ──────────────────────────────────────────────
  idea-list        {count} {subtopic} Ideas for {year}
  checklist        The Ultimate {subtopic} Checklist
  comparison       {A} vs {B}: Which Is Better?
  ... 7 more

Step 6: Generate Pages

Start a generation job that combines your project, selected niches, and schema:

contento collections generate \
    --project proj_8f3a... \
    --schema "idea-list" \
    --niches "niche_01...,niche_02..." \
    --workers 5

  Generating 38 pages across 2 niches...

  [00:04] ██▁▁▁▁▁▁▁▁▁▁▁▁▁▁  5/38  (13%)
  [00:12] ███████▁▁▁▁▁▁▁▁▁ 20/38  (53%)
  [00:20] ████████████████ 38/38 (100%)

  Validation: 36 passed, 2 retried, 0 failed
  Collection: col_9d2f...

The engine expands subtopics from each niche, deduplicates titles, generates JSON content via AI, validates against the schema, and retries failures automatically.

Step 7: Publish

Once generation completes, publish the pages to your subdomain:

contento publish \
    --collection col_9d2f... \
    --mode immediate

  Publishing 36 pages...
  ✓ Internal linking pass: 108 cross-links added
  ✓ Sitemap rebuilt: 36 URLs
  ✓ Schema markup injected
  ✓ OG meta tags generated

  Published to: resources.example.com

Your pages are now live. Visit your subdomain to see them:

  • https://resources.example.com/100-api-testing-ideas-2026
  • https://resources.example.com/100-cicd-pipeline-ideas-2026
  • https://resources.example.com/sitemap.xml

What Happened Behind the Scenes

During the generate and publish steps, Contento:

  1. Expanded subtopics — Pulled 10–20 subtopics from each selected niche
  2. Deduplicated — Removed overlapping subtopics across niches
  3. Built prompts — Combined niche context + schema template + validation rules
  4. Generated JSON — Sent prompts to AI provider with concurrent workers
  5. Validated — Checked item counts, string lengths, enum values, required fields
  6. Retried failures — Adjusted prompt and retried once for failed pages
  7. Rendered HTML — Transformed validated JSON through the schema's renderer
  8. Applied chrome — Wrapped each page in your header/footer/CSS
  9. Cross-linked — Added internal links between related pages in the same niche
  10. Built sitemap — Generated sitemap.xml with all published URLs
