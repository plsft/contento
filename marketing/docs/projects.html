Projects & Domains

Every pSEO campaign starts with a project tied to a subdomain.

What Is a pSEO Project?

A project is the top-level unit in Contento pSEO. It maps to a single subdomain (e.g., resources.example.com) and owns all the collections, pages, chrome, and analytics data generated under that domain. You can run multiple projects from a single Contento installation, each targeting different audiences or brands.

Each project stores:

  • Domain and subdomain — Where generated pages are served
  • Custom chrome — Header HTML, footer HTML, and CSS that wrap every page
  • Back-link and CTA configuration — Links injected into generated content
  • Collections — Generation jobs that produce pages
  • Analytics — GSC data, click/impression metrics, and performance rankings

Creating a Project

Via the Admin UI

  1. Navigate to Admin → pSEO → Projects.
  2. Click New Project.
  3. Enter a project name, domain, and subdomain.
  4. Optionally upload header/footer HTML and CSS.
  5. Save. The project starts in pending_dns status.

Via the API

curl -X POST https://app.contentocms.com/api/v1/pseo/projects \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "DevTools Resources",
    "domain": "example.com",
    "subdomain": "resources.example.com",
    "backlinkUrl": "https://example.com",
    "backlinkAnchorText": "Example Inc",
    "ctaHtml": "<a href=\"https://example.com/signup\" class=\"cta-btn\">Start Free Trial</a>"
  }'

DNS & CNAME Setup

After creating a project, you need to point the subdomain to Contento:

  1. Log in to your DNS provider (Cloudflare recommended).
  2. Add a CNAME record:
    TypeNameTargetProxy
    CNAMEresourcespseo.contentocms.comProxied (orange cloud)
  3. Wait for DNS propagation (usually seconds with Cloudflare).
  4. Contento detects the CNAME and transitions the project from pending_dns to active.

SSL Configuration

When using Cloudflare as your DNS provider, SSL is handled automatically. Set your Cloudflare SSL/TLS mode to Full (strict). No certificates need to be provisioned or configured in Contento — Cloudflare terminates TLS at the edge and proxies to your origin over HTTPS.

Custom Chrome

Chrome is the header, footer, and CSS that wraps every generated page in your project. This is how you make pSEO pages match your main site's look and feel.

  • Header HTML — Navigation bar, logo, site-wide banner
  • Footer HTML — Footer links, copyright, social icons
  • Custom CSS — Styles scoped to your chrome elements

Upload chrome via the admin UI on the project edit page, or via the API:

curl -X PUT https://app.contentocms.com/api/v1/pseo/projects/{id}/chrome \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "headerHtml": "<nav class=\"site-nav\"><a href=\"/\">Home</a></nav>",
    "footerHtml": "<footer>&copy; 2026 Example Inc</footer>",
    "customCss": ".site-nav { padding: 1rem 2rem; background: #fff; border-bottom: 1px solid #e5e7eb; }"
  }'

See Custom Chrome for the full guide.

Back-link & CTA Configuration

Every project can inject back-links and calls to action into generated content:

  • Back-link URL — The target URL for contextual links in generated content (e.g., your homepage)
  • Back-link Anchor Text — The text for the link (e.g., your brand name)
  • CTA HTML — A raw HTML snippet injected at strategic points in pages (e.g., a signup button)

These are configured at the project level and automatically applied to all pages generated under the project.

Project Lifecycle

StatusDescription
pending_dns Project created, waiting for CNAME verification. Pages are not served.
active DNS verified. Pages are live and served on the subdomain. Generation and analytics are enabled.
suspended Manually paused by the admin. Pages return a 503. No generation or analytics sync runs.

Transition between states via the admin UI or the API:

# Suspend a project
curl -X POST https://app.contentocms.com/api/v1/pseo/projects/{id}/suspend \
  -H "Authorization: Bearer <token>"

# Reactivate a suspended project
curl -X POST https://app.contentocms.com/api/v1/pseo/projects/{id}/activate \
  -H "Authorization: Bearer <token>"
