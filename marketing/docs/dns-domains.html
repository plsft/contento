DNS & Domains

CNAME setup, Cloudflare configuration, SSL termination, and DNS verification.

How Domain Routing Works

Contento serves generated pages on your subdomain (e.g., articles.example.com) using CNAME-based routing. When a visitor requests a page:

  1. DNS resolves articles.example.com to pseo.contentocms.com (via your CNAME)
  2. The request arrives at Contento's cloud infrastructure with the Host header set to your subdomain
  3. Contento looks up the project associated with that subdomain
  4. The page is served with your custom chrome (header/footer/CSS)

CNAME Setup

Add a single CNAME record in your DNS provider:

FieldValue
TypeCNAME
NameYour subdomain prefix (e.g., articles, resources, tips)
Targetpseo.contentocms.com
TTLAuto (or 300 seconds)
Important: Do not use an A record. CNAME is required so that Contento can route traffic based on the Host header. An A record pointing directly to an IP will not work for multi-tenant subdomain routing.

Cloudflare Configuration

Cloudflare is the recommended DNS provider because it handles SSL automatically. Configure these settings:

DNS Record

In Cloudflare's DNS settings, add the CNAME record with the orange cloud (Proxied) enabled. This routes traffic through Cloudflare's CDN and enables SSL.

# Cloudflare DNS panel
Type:    CNAME
Name:    articles
Target:  pseo.contentocms.com
Proxy:   Proxied (orange cloud)
TTL:     Auto

SSL/TLS Mode

Set your Cloudflare SSL/TLS encryption mode to Full (strict):

  1. Go to SSL/TLS in your Cloudflare dashboard
  2. Select Full (strict)

This ensures end-to-end encryption. Cloudflare terminates TLS at the edge and proxies to Contento over HTTPS.

Page Rules (Optional)

For additional control, you can add Cloudflare page rules for your subdomain:

  • Cache Level: Standard — Caches static assets served by Contento
  • Browser Cache TTL: 4 hours — Reduces origin requests
  • Always Use HTTPS — Redirects HTTP to HTTPS

Non-Cloudflare DNS Providers

If you use a different DNS provider (e.g., Route 53, Namecheap, GoDaddy), the CNAME setup is the same. Contento's cloud infrastructure handles SSL automatically for all verified subdomains, so no additional SSL configuration is required on your end.

Recommended: Even if your registrar is elsewhere, using Cloudflare as your DNS provider (free tier) adds CDN caching and DDoS protection on top of the SSL that Contento already provides.

DNS Verification

After you add the CNAME record, Contento automatically verifies it. The verification process:

  1. Contento polls DNS every 5 minutes for new projects in pending_dns status
  2. A DNS lookup checks that the subdomain's CNAME points to pseo.contentocms.com
  3. On success, the project transitions from pending_dns to active
  4. Pages can now be served on the subdomain
# Check verification status
$ contento projects status --project proj_8f3a...

  Project:    DevTools Resources
  Subdomain:  resources.example.com
  DNS Status: verified
  SSL:        active
  Pages:      248 published

Manual Verification

If you need to trigger verification immediately (e.g., you just added the CNAME and do not want to wait):

$ contento projects verify --project proj_8f3a...

  Checking DNS for resources.example.com...
  ✓ CNAME verified: points to pseo.contentocms.com
  ✓ SSL certificate active
  Project status: active

Multiple Subdomains

You can run multiple subdomains under the same root domain, each as a separate Contento project:

SubdomainProjectContent Focus
articles.example.comArticles ProjectIdea lists, how-to guides
resources.example.comResources ProjectComparisons, alternatives
learn.example.comLearning ProjectFAQs, glossaries

Each project has its own niches, collections, chrome, and analytics. Add a separate CNAME record for each subdomain.

Troubleshooting

ProblemCauseSolution
DNS not verifying CNAME not propagated yet Wait up to 5 minutes (Cloudflare) or 48 hours (other providers). Use dig or nslookup to check.
SSL errors Cloudflare SSL mode wrong Set to Full (strict), not Flexible.
Pages return 404 Project not active Check project status with contento projects status — must be active.
Wrong pages served Subdomain mismatch Ensure the subdomain in your project exactly matches the CNAME name + domain.
