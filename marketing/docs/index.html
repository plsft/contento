Quick Start

Get Contento pSEO generating pages in under five minutes.

What is Contento pSEO?

Contento pSEO is a programmatic SEO engine that generates thousands of high-quality, niche-targeted pages using AI and structured content schemas. Unlike a traditional CMS where you write each page by hand, Contento automates the entire pipeline: you define a content schema (e.g., "10 Best Tools for {subtopic}"), select your target niches, and the engine cross-multiplies niches with subtopics to produce unique, validated pages at scale.

Contento is not a CMS. It is a purpose-built tool for programmatic SEO. Every generated page includes structured JSON-LD, deterministic title patterns for SERP targeting, custom chrome (your branding header/footer), and a feedback loop from Google Search Console analytics. The result is a self-improving content engine that identifies what works, doubles down on performing niches, and prunes underperformers automatically.

Prerequisites

Dependency Version Required
Docker + Docker Compose 24+ / v2 Option A
.NET SDK 10.0 or later Option B
PostgreSQL 17 Yes
Redis 7.x Yes (job queue)

Install & Run

Option A — Docker (recommended)

# Clone the repository
git clone https://github.com/plsft/contento.git
cd contento

# Start all services (app, PostgreSQL 17, Redis 7)
docker-compose up -d

The application starts at https://localhost:5001. The admin panel is at /admin.

Option B — .NET SDK

git clone https://github.com/plsft/contento.git
cd contento
dotnet restore
cd src/Contento.Web
dotnet run

Ensure PostgreSQL 17 and Redis 7 are running locally and configured via environment variables (see below).

Tip: On first startup, Contento automatically runs database migrations via Bowtie, creates all tables, and seeds 150 niches and 10 content schemas. No manual migration step is needed.

First Project Setup

  1. Navigate to Admin → pSEO → Setup Wizard.
  2. Enter your domain (e.g., example.com) and subdomain (e.g., resources.example.com).
  3. Add a CNAME record in your DNS provider pointing the subdomain to Contento's app domain.
  4. Choose your target niches (e.g., Developer Tools, SaaS Analytics) and content types (e.g., idea-list, comparison).
  5. Click Generate First Batch to kick off your first page generation run.

The generation pipeline expands subtopics, deduplicates, queues AI generation jobs, validates the output, and publishes. You can monitor real-time progress via the SSE streaming endpoint in the admin dashboard.

DATABASE_CONNECTION_STRING PostgreSQL 17 connection string Required
REDIS_CONNECTION_STRING Redis 7 connection string for job queue and caching Required
CONTENTO_ADMIN_PASSWORD Password for the default admin account Required
GSC_CLIENT_ID Google Search Console OAuth client ID Empty (analytics disabled)
GSC_CLIENT_SECRET Google Search Console OAuth client secret Empty (analytics disabled)
ASPNETCORE_ENVIRONMENT Runtime environment Production
ASPNETCORE_URLS Listening URLs http://+:5000
