Docs API Reference

API Reference

Complete REST API documentation for the Contento CMS.

Overview

Contento exposes a comprehensive REST API with 135 endpoints across multiple groups. All endpoints include OpenAPI metadata and are documented in the interactive Scalar API explorer.

  1. Authentication API (/api/v1/auth) — Token-based authentication
  2. Content Delivery API (/api/v1/content) — Public, read-only endpoints for frontend consumption
  3. Management API (/api/v1) — Authenticated CRUD endpoints for managing content
  4. Newsletter API (/api/v1/newsletter) — Subscriber and campaign management
  5. Membership API (/api/v1/membership) — Stripe-powered subscriptions and billing
  6. AI API (/api/v1/ai) — AI-powered content generation and analysis
  7. Sites API (/api/v1/sites) — Multi-site management
  8. Scheduler API (/api/v1/scheduler) — Task scheduling and automation
  9. Redirects API (/api/v1/redirects) — URL redirect management with hit tracking
  10. MCP Integration — Guide for using the API with AI agents via Model Context Protocol

Base URL

https://your-contento-site.com/api/v1

Response format

All successful responses follow this envelope:

{
  "data": { ... }            // Single object or array
}

// With pagination:
{
  "data": [ ... ],
  "meta": {
    "page": 1,
    "pageSize": 20,
    "totalCount": 42,
    "totalPages": 3
  }
}

Error format

{
  "error": {
    "code": "NOT_FOUND",
    "message": "Post not found."
  }
}

Pagination

Paginated endpoints accept these query parameters:

ParameterTypeDefaultDescription
pageinteger1Page number (1-indexed)
pageSizeinteger20Items per page

Authentication

Management API endpoints require a bearer token in the Authorization header:

Authorization: Bearer <your-token>

Tokens are HMAC-SHA256 signed and expire after 24 hours. See the Authentication API below.

Authentication API

Base path: /api/v1/auth — All endpoints are public (no token required to call).

POST /api/v1/auth/token

Exchange email and password for a bearer token.

Request body

{
  "email": "admin@contento.local",
  "password": "your-password"
}

Response 200 OK

{
  "data": {
    "token": "eyJVc2VySWQ...signature",
    "tokenType": "Bearer",
    "expiresAt": "2026-02-19T12:00:00Z",
    "expiresIn": 86400,
    "user": {
      "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "email": "admin@contento.local",
      "displayName": "Admin",
      "role": "owner"
    }
  }
}

Error responses

StatusCodeDescription
400INVALID_REQUESTEmail and password are required.
401INVALID_CREDENTIALSInvalid email or password.
401ACCOUNT_DISABLEDThis account has been disabled.
POST /api/v1/auth/refresh

Refresh an existing valid token. Returns a new token with a fresh 24-hour expiry.

Request body

{
  "token": "eyJVc2VySWQ...signature"
}

Response 200 OK

{
  "data": {
    "token": "eyJVc2VySWQ...newSignature",
    "tokenType": "Bearer",
    "expiresAt": "2026-02-20T12:00:00Z",
    "expiresIn": 86400
  }
}

Error responses

StatusCodeDescription
400INVALID_REQUESTToken is required.
401INVALID_TOKENToken is invalid or expired.
GET /api/v1/auth/me

Validate a token and return the associated user profile. Requires Authorization: Bearer <token> header.

Response 200 OK

{
  "data": {
    "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "email": "admin@contento.local",
    "displayName": "Admin",
    "role": "owner",
    "isActive": true
  }
}

Error responses

StatusCodeDescription
401NO_TOKENBearer token required.
401INVALID_TOKENToken is invalid or expired.
404NOT_FOUNDUser not found.
POST /api/v1/auth/register

Register a new user account. Creates a viewer-role user and sends an email verification link.

Request body

{
  "email": "user@example.com",
  "password": "securePassword123",
  "displayName": "Jane Doe"
}

Response 201 Created

{
  "data": { "message": "Account created. Please check your email to verify." }
}
POST /api/v1/auth/forgot-password

Request a password reset email. Always returns success to prevent email enumeration.

Request body

{
  "email": "user@example.com"
}
POST /api/v1/auth/reset-password

Reset a user’s password using a valid reset token (1-hour expiry).

Request body

{
  "token": "reset-token-from-email",
  "newPassword": "newSecurePassword456"
}
POST /api/v1/auth/verify-email

Verify a user’s email address using the token sent during registration.

Request body

{
  "token": "verification-token-from-email"
}

GET /api/v1/posts/{id}

Get a single post by ID (any status).

Response 200 OK

{
  "data": { /* full Post object */ }
}
GET /api/v1/posts/by-slug/{slug}

Get a single post by slug (any status). Useful for admin preview of draft posts.

PUT /api/v1/posts/{id}

Update an existing post. Only send the fields you want to change (partial update).

Request body (all fields optional)

{
  "title": "Updated Title",
  "subtitle": "Updated subtitle",
  "slug": "updated-slug",
  "bodyMarkdown": "# Updated content",
  "coverImageUrl": "/uploads/new-cover.jpg",
  "status": "published",
  "visibility": "public",
  "featured": true,
  "metaTitle": "Updated SEO Title",
  "metaDescription": "Updated SEO description",
  "canonicalUrl": "https://example.com/original",
  "tags": ["updated", "tags"]
}
Auto-publish: Setting status to "published" on a draft post automatically sets publishedAt to the current UTC timestamp.
DELETE /api/v1/posts/{id}

Permanently delete a post. Returns 204 No Content on success.

POST /api/v1/posts/{id}/publish

Publish a draft or scheduled post. Sets status to published and publishedAt to now.

Response 200 OK

{
  "data": { /* updated Post object with status: "published" */ }
}
POST /api/v1/posts/{id}/unpublish

Revert a published post to draft. Removes it from the public site.

Response 200 OK

{
  "data": { /* updated Post object with status: "draft" */ }
}

MCP Integration

Contento's REST API is designed for Model Context Protocol (MCP) compatibility. AI agents can use the full API surface to autonomously create, manage, and publish content.

Why Contento works with MCP

  • Stateless bearer tokens — No cookies or sessions. Agents authenticate once and include the token in every request.
  • Full OpenAPI spec — Available at /openapi/v1.json. Agents can discover all 135 endpoints, request/response schemas, and parameter types automatically.
  • Structured JSON responses — Every endpoint returns a consistent { data, meta } envelope that agents can parse reliably.
  • Complete CRUD coverage — Posts, categories, media, menus, themes, layouts, comments, redirects — every content type has full API support.
  • Built-in AI endpoints — Text completion, streaming, and generative theme/layout creation are available as first-party API calls.

Agent workflow example

A typical MCP agent workflow for content generation:

// 1. Authenticate — get a bearer token
POST /api/v1/auth/token
{ "email": "agent@site.com", "password": "..." }
// → { data: { token: "eyJ...", expiresIn: 86400 } }

// 2. Create a post
POST /api/v1/posts
Authorization: Bearer eyJ...
{
  "title": "10 Tips for Better SEO",
  "bodyMarkdown": "# Introduction\n\n...",
  "tags": ["seo", "marketing"],
  "metaTitle": "10 SEO Tips for 2026",
  "metaDescription": "Actionable SEO tips...",
  "status": "draft"
}
// → { data: { id: "a1b2c3...", slug: "10-tips-for-better-seo" } }

// 3. Upload a cover image
POST /api/v1/media/upload  (multipart/form-data)
// → { data: { storagePath: "/uploads/cover.webp" } }

// 4. Attach cover image and publish
PUT /api/v1/posts/a1b2c3...
{ "coverImageUrl": "/uploads/cover.webp" }

POST /api/v1/posts/a1b2c3.../publish
// → 200 OK — post is live
