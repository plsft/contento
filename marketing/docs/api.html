Docs API Reference

API Reference

All 39 pSEO endpoints grouped by resource.

Overview

The Contento pSEO API is organized into 7 endpoint groups with a total of 39 endpoints. All endpoints are authenticated via bearer token unless noted otherwise.

Base URL

https://app.contentocms.com/api/v1

Response Format

{
  "data": { ... }            // Single object or array
}

// With pagination:
{
  "data": [ ... ],
  "meta": {
    "page": 1,
    "pageSize": 20,
    "totalCount": 42,
    "totalPages": 3
  }
}

Error Format

{
  "error": {
    "code": "NOT_FOUND",
    "message": "Resource not found."
  }
}

Authentication

All API requests require a bearer token in the Authorization header:

Authorization: Bearer <your-token>

Tokens are HMAC-SHA256 signed and expire after 24 hours.

POST /api/v1/auth/token

Exchange email and password for a bearer token.

Request

{
  "email": "admin@contento.local",
  "password": "your-password"
}

Response 200 OK

{
  "data": {
    "token": "eyJVc2VySWQ...signature",
    "tokenType": "Bearer",
    "expiresAt": "2026-03-12T12:00:00Z",
    "expiresIn": 86400
  }
}
POST /api/v1/auth/refresh

Refresh an existing valid token. Returns a new token with a fresh 24-hour expiry.

Request

{
  "token": "eyJVc2VySWQ...signature"
}

Response 200 OK

{
  "data": {
    "token": "eyJVc2VySWQ...newSignature",
    "tokenType": "Bearer",
    "expiresAt": "2026-03-13T12:00:00Z",
    "expiresIn": 86400
  }
}

Projects (8 endpoints)

POST /api/v1/pseo/projects

Create a new pSEO project.

Request

{
  "name": "DevTools Resources",
  "domain": "example.com",
  "subdomain": "resources.example.com",
  "backlinkUrl": "https://example.com",
  "backlinkAnchorText": "Example Inc",
  "ctaHtml": "<a href='https://example.com/signup'>Start Free Trial</a>"
}

Response 201 Created

{
  "data": {
    "id": "proj-uuid",
    "name": "DevTools Resources",
    "subdomain": "resources.example.com",
    "status": "pending_dns",
    "createdAt": "2026-03-11T10:00:00Z"
  }
}
GET /api/v1/pseo/projects

List all projects. Supports pagination.

GET /api/v1/pseo/projects/{id}

Get a single project by ID, including chrome and configuration.

PUT /api/v1/pseo/projects/{id}

Update project settings (name, backlink, CTA, etc.).

DELETE /api/v1/pseo/projects/{id}

Delete a project and all its collections, pages, and analytics data.

PUT /api/v1/pseo/projects/{id}/chrome

Upload or update custom chrome (header HTML, footer HTML, CSS).

Request

{
  "headerHtml": "<nav>...</nav>",
  "footerHtml": "<footer>...</footer>",
  "customCss": "nav { padding: 1rem; }"
}
POST /api/v1/pseo/projects/{id}/activate

Activate or reactivate a suspended project.

POST /api/v1/pseo/projects/{id}/suspend

Suspend a project. Pages return 503, generation and analytics are paused.

Niches (5 endpoints)

GET /api/v1/pseo/niches

List all niches. Filter by ?category=Software or ?isSystem=true.

GET /api/v1/pseo/niches/{id}

Get a single niche with full context JSON (audience, pain_points, subtopics, etc.).

POST /api/v1/pseo/niches

Create a custom niche with full context definition.

Request

{
  "name": "Developer Tools",
  "slug": "developer-tools",
  "category": "Software / SaaS",
  "context": {
    "audience": "Software developers and DevOps engineers",
    "pain_points": ["Too many tools", "Integration complexity"],
    "monetization": ["Freemium", "Per-seat SaaS"],
    "content_that_works": ["Comparison posts", "Tool roundups"],
    "subtopics": ["CI/CD pipelines", "API testing tools", "..."]
  }
}
PUT /api/v1/pseo/niches/{id}

Update a custom niche. System niches are read-only.

DELETE /api/v1/pseo/niches/{id}

Delete a custom niche. System niches cannot be deleted.

Schemas (5 endpoints)

GET /api/v1/pseo/schemas

List all content schemas with title patterns and renderer slugs.

GET /api/v1/pseo/schemas/{id}

Get a schema by ID with full SchemaJson and PromptTemplate.

POST /api/v1/pseo/schemas

Create a custom content schema.

Request

{
  "name": "Idea List",
  "slug": "idea-list",
  "titlePattern": "{count} {subtopic} Ideas for {year}",
  "metaDescPattern": "Discover {count} actionable {subtopic} ideas...",
  "rendererSlug": "idea-list",
  "promptTemplate": "Generate a JSON object with...",
  "schemaJson": { /* JSON Schema definition */ }
}
PUT /api/v1/pseo/schemas/{id}

Update a content schema.

DELETE /api/v1/pseo/schemas/{id}

Delete a schema. Fails if used by active collections.

Collections (10 endpoints)

POST /api/v1/pseo/collections

Create a new collection (schema + niches + URL pattern + publishing mode).

Request

{
  "projectId": "proj-uuid",
  "schemaId": "schema-uuid",
  "nicheIds": ["niche-uuid-1", "niche-uuid-2"],
  "urlPattern": "/{niche}/{subtopic}-ideas-{year}",
  "titleTemplate": "{count} {subtopic} Ideas for {year}",
  "publishingMode": "immediate",
  "concurrentWorkers": 10
}
GET /api/v1/pseo/collections

List collections. Filter by ?projectId=.

GET /api/v1/pseo/collections/{id}

Get collection details including page counts and generation status.

PUT /api/v1/pseo/collections/{id}

Update collection settings (URL pattern, publishing mode, workers).

DELETE /api/v1/pseo/collections/{id}

Delete a collection and all its generated pages.

POST /api/v1/pseo/collections/{id}/generate

Start or restart the generation pipeline for this collection.

Response 202 Accepted

{
  "data": {
    "jobId": "job-uuid",
    "totalPages": 40,
    "status": "queued"
  }
}
POST /api/v1/pseo/collections/{id}/pause

Pause an in-progress generation. Queued jobs are preserved.

POST /api/v1/pseo/collections/{id}/resume

Resume a paused generation.

GET /api/v1/pseo/collections/{id}/progress

SSE stream of real-time generation progress. Returns text/event-stream.

Event format

data: {"total": 40, "completed": 12, "failed": 1, "status": "generating"}
data: {"total": 40, "completed": 40, "failed": 2, "status": "complete"}
POST /api/v1/pseo/collections/{id}/publish

Publish all draft pages in the collection (for manual publishing mode).

Pages (7 endpoints)

GET /api/v1/pseo/pages

List pages. Filter by ?projectId=, ?collectionId=, ?nicheId=, ?status=.

GET /api/v1/pseo/pages/{id}

Get a single page with full content JSON and rendered HTML.

PUT /api/v1/pseo/pages/{id}

Update a page's content JSON or metadata. Triggers re-rendering.

DELETE /api/v1/pseo/pages/{id}

Delete a single page.

POST /api/v1/pseo/pages/{id}/publish

Publish a single draft page.

POST /api/v1/pseo/pages/{id}/unpublish

Revert a published page to draft status.

POST /api/v1/pseo/pages/{id}/regenerate

Re-run AI generation for a single page with an updated prompt.

Analytics (9 endpoints)

GET /api/v1/pseo/analytics/overview

Project-level summary: total clicks, impressions, average CTR, average position, indexed count.

Query parameters

ParameterTypeDescription
projectIduuidRequired. The project to get analytics for.

Response 200 OK

{
  "data": {
    "totalClicks": 12450,
    "totalImpressions": 285000,
    "averageCtr": 0.0437,
    "averagePosition": 16.3,
    "indexedPages": 980,
    "totalPages": 1024
  }
}
GET /api/v1/pseo/analytics/pages

Page-level performance data. Paginated and sortable by clicks, impressions, CTR, or position.

GET /api/v1/pseo/analytics/niches

Niche-level aggregated performance: total clicks, pages, average position per niche.

GET /api/v1/pseo/analytics/queries

Top search queries driving traffic to the project's pages.

GET /api/v1/pseo/analytics/zero-traffic

Pages with zero clicks for 60+ consecutive days. Used for underperformer detection.

GET /api/v1/pseo/analytics/export

CSV export of analytics data. Use ?format=csv.

POST /api/v1/pseo/analytics/gsc/connect

Initiate Google Search Console OAuth connection. Returns an authorization URL.

Response 200 OK

{
  "data": {
    "authorizationUrl": "https://accounts.google.com/o/oauth2/auth?..."
  }
}
POST /api/v1/pseo/analytics/gsc/sync

Trigger a manual GSC data sync (normally runs daily as a background job).

GET /api/v1/pseo/analytics/gsc/status

Check GSC connection status and last sync timestamp.

Response 200 OK

{
  "data": {
    "connected": true,
    "property": "sc-domain:example.com",
    "lastSyncAt": "2026-03-11T02:00:00Z",
    "lastSyncStatus": "success"
  }
}
Endpoint count: 2 (auth) + 8 (projects) + 5 (niches) + 5 (schemas) + 10 (collections) + 7 (pages) + 9 (analytics) = 46 total endpoints. The 39 pSEO-specific endpoints exclude the 2 auth and 5 schema endpoints that are shared infrastructure.
